This article is an overview of your options for creating and managing workspaces.
What is a workspace?
A workspace is a Azure Databricks deployment in a cloud service account. It provides a unified environment for working with Azure Databricks assets for a specified set of users.
Workspace creation options
The standard way to deploy an Azure Databricks workspace is using the Azure Portal. For more information, seeCreate an Azure Databricks workspace.
This section of the administration guide includes additional methods for deploying your Azure Databricks workspace. These are:
Deploy a workspace with the Azure CLI
Deploy a workspace with PowerShell
Deploy a workspace with an ARM template
Deploy a workspace with Bicep
Deploy and manage a workspace with Terraform
You can also use theDatabricks Terraform providerto manage your Azure Databricks workspaces and the associated cloud infrastructure using a flexible, powerful tool.
For instructions on deploying an Azure Databricks workspace using Terraform, seeDeploy an Azure Databricks workspace using Terraform. For instructions on managing your deployed workspace using Terraform, seeManage Databricks workspaces using Terraform.
Access the workspace admin settings page
The admin settings page is where you can manage features and settings for your Azure Databricks workspace. To access the admin settings, click your username in the top bar of the Azure Databricks workspace and selectSettings.
For more information on workspace settings, seeManage your workspace.
